# 代表者ページ リデザイン設計書

## 概要

**目的**: 求人応募者と関係機関の両方に響く、親しみやすく個性的なマガジン風ページ

**ターゲット**:
- 求人応募者: 「この人と働きたい」と思える
- 関係機関: 代表の人柄・方針が伝わり信頼を得る

**トーン**: 親しみやすさ + ユニーク（堅すぎず、軽すぎず）

---

## デザインコンセプト

### スタイル: マガジン風インタビュー記事

読み物として楽しめる構成。代表への「インタビュー形式」で語りかけるように展開。

### カラーパレット

```css
/* メイン */
--color-paper: #f7f2ea      /* 背景：紙の質感 */
--color-ink: #2b2b2b        /* テキスト */
--color-ink-soft: #3a3a3a   /* サブテキスト */

/* アクセント */
--color-olive: #2f5b4a      /* 見出し・強調 */
--color-logo-light-green: #7FC5A0  /* ブランドカラー */
--color-logo-yellow: #F4E951       /* ハイライト */
```

### タイポグラフィ

- **見出し**: `heading-mincho`（明朝体） + Fluid変数
- **本文**: Noto Sans JP（ゴシック体）
- **引用・強調**: イタリック風の斜体表現

## ページ構成（新規）

### セクション構成

```
1. ヒーロー
   - 代表写真 + キャッチフレーズ
   - 「完璧より最適を。」

2. 自己紹介（インタビュー形式）
   Q: 浅井さんってどんな人？
   A: ストレングスファインダーの結果を絡めて自己紹介

3. 仕事への向き合い方
   Q: 仕事で大切にしていることは？
   A: 効率・改善・仕組み化への情熱を親しみやすく

4. チームとの関わり方
   Q: どんなチームを作りたい？
   A: 相性の良い人・一緒に働きたい人の特徴

5. フラクタルへの想い
   Q: フラクタルで実現したいことは？
   A: ビジョン・将来像

6. 代表からのメッセージ
   - 採用候補者・関係機関へ向けた温かいメッセージ

7. CTA
   - 会社情報へ
   - 採用情報へ
```

**コンテンツ案**:

ひとことで言うと「仕組みづくりが好きな人」です。

ストレングスファインダーという才能診断では、
TOP5が「個別化・着想・戦略性・活発性・分析思考」でした。

つまり...

- **一人ひとりの違いを見抜く力** がある
- **新しいアイデアをつなげる** のが得意

- **考えるより先に動く** タイプ
- **「なぜ？」を掘り下げる** のが癖

訪問看護の現場でも、「この人にはこのケアが合う」

常に最適解を探しています。

### 3. 仕事への向き合い方セクション

**コンテンツ案**:

```markdown
## 仕事で大切にしていることは？

### 「なぜ？」から始める

「昔からこうだから」は理由になりません。
すべてに理由があるべきで、理由がないなら変えるべき。

### 最短距離を探す

遠回りは嫌い。無駄な会議、意味のない書類、形だけの手続き。
「それ、本当に必要？」と常に問いかけます。

### 動きながら考える

完璧な計画を待っていたら、何も始まりません。
まず動いて、やりながら修正する。
このスピード感がフラクタルの強みです。
```

**ビジュアル要素**:
- フローチャート: 「Input → Logic → Action → Feedback」のサイクル
- カード形式で3つの価値観を表示

### 4. チームとの関わり方セクション

**コンテンツ案**:

```markdown
## どんなチームを作りたい？

### 一緒に働きたい人

- 「なぜ？」を一緒に追求できる人
- 自分で考えて動ける人
- 改善提案を恐れない人
- 論理的に議論できる人

### こんな人は難しいかも

- 「言われたことだけ」で満足する人
- 変化を嫌う人
- 環境のせいにする人

正直に書いていますが、これは「合う・合わない」の話。
どちらが良い・悪いではありません。

フラクタルが合う人には、最高の環境を用意したい。
それが私のスタンスです。
```

**ビジュアル**:
- 2カラムで「合う人」「難しいかも」を並べる
- アイコンでわかりやすく

### 5. フラクタルへの想いセクション

**コンテンツ案**:

```markdown
## フラクタルで実現したいことは？

訪問看護の業界には、まだまだ「非効率」が残っています。

紙の書類、無駄な移動、属人的なケア...

これらを「仕組み」に変えて、
看護師さんが「看護」に集中できる環境を作りたい。

そして、その仕組みが回り始めたら、
私は次の課題を見つけに行く。

「完成したら次へ」

それが私のスタイルであり、
フラクタルを常に進化させ続ける原動力です。
```

## コンポーネント設計

### 1. QABlock（Q&A形式ブロック）

```tsx
interface QABlockProps {
  question: string;
  children: React.ReactNode;
}

// デザイン
// Q: 大きな質問文（明朝体）
// A: 回答コンテンツ（カード内）
```

### 2. StrengthCard（資質カード）

```tsx
interface StrengthCardProps {
  rank: number;
  name: string;
  nameEn: string;
  description: string;
}

// デザイン
// - 番号バッジ（#1, #2...）
// - 資質名（日本語 + 英語）
// - 説明文
// - ホバーで拡大
```

### 3. ValueCard（価値観カード）

```tsx
interface ValueCardProps {
  icon: string;
  title: string;
  description: string;
}

// デザイン
// - アイコン（絵文字 or SVG）
// - タイトル
// - 説明文
```

### 4. CompatibilityList（相性リスト）

```tsx
interface CompatibilityListProps {
  type: 'good' | 'challenging';
  items: string[];
}

// デザイン
// good: 緑系ボーダー、チェックアイコン
// challenging: オレンジ系ボーダー、注意アイコン
```

## 技術仕様

### 使用する既存要素

| 要素 | 用途 |
|------|------|
| `body-editorial` | 背景グラデーション |
| `heading-mincho` | 見出し |
| `--color-paper`, `--color-ink` | カラー |
| `rounded-2xl`, `rounded-3xl` | カード角丸 |
| Fluid変数 | レスポンシブ対応 |
| FadeIn アニメーション | スクロール演出 |

### 新規作成するコンポーネント

1. `QABlock` - Q&A形式のセクションブロック
2. `StrengthCard` - ストレングスファインダー資質カード
3. `ValueCard` - 価値観カード
4. `CompatibilityList` - 相性リスト

### レスポンシブ対応

- Mobile: シングルカラム、資質カード横スクロール
- Tablet: 2カラム
- Desktop: 2カラム（黄金比）
