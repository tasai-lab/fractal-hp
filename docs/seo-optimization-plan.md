## 現状分析（フラクタル訪問看護）

### 強み
- 構造化データが充実（LocalBusiness, FAQPage, JobPosting, BreadcrumbList）
- 地域別求人ページが既存（/recruit/areas/[slug]）
- メタデータが適切に設定済み

### 改善が必要な点
1. タイトルに【地域名】形式が未使用
2. 採用ページのh1に地域名がない
3. サービス利用者向け地域別ページがない
4. Serviceスキーマが未実装

## 改善提案

### 1. トップページ最適化
**現在のタイトル:**
```
フラクタル訪問看護 船橋 | 船橋市・八千代市・習志野市の訪問看護ステーション
```

**改善案:**
```
【船橋市・八千代市・習志野市】訪問看護ならフラクタル | 24時間365日対応
```

**改善理由:**
- 【】で地域名を囲む（競合で有効）
- 「なら」で検索意図に対応
- 差別化ポイント（24時間365日）を明記

### 2. 採用ページ最適化
**現在のタイトル:**
```
船橋市・八千代市・習志野市の訪問看護求人 | フラクタル訪問看護
```

**改善案:**
```
【船橋市・八千代市・習志野市】訪問看護師・PT・OT・ST求人 | フラクタル訪問看護
```

### 3. 地域別ランディングページ新設
```
/areas/funabashi       → 船橋市の訪問看護
/areas/yachiyo         → 八千代市の訪問看護
/areas/narashino       → 習志野市の訪問看護
/areas/chiba-hanamigawa → 千葉市花見川区の訪問看護
```

各ページに含めるコンテンツ:
- h1: 【{地域名}】訪問看護ならフラクタル
- 地域の特性・高齢化率情報
- 訪問可能エリア（町名リスト）
- 地域特化FAQ
- LocalBusiness構造化データ（地域別）

### 4. 構造化データ追加
- Serviceスキーマ（サービス内容詳細）
- 地域別LocalBusinessスキーマ
- HowToスキーマ（ご利用の流れ）

### 5. 内部リンク構造改善
- フッターに地域別ページへのリンク追加
- 地域別ページ間の相互リンク
- パンくずリストUIの表示
